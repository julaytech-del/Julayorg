Security

Your data is safe with us

We apply enterprise-grade security practices to protect every workspace, task, and team member on Julay.

🔐

Encryption everywhere

All data is encrypted in transit (TLS 1.3) and at rest (AES-256). No exceptions.

🏗️

Isolated infrastructure

Each workspace runs in a logically isolated environment on AWS with strict network policies.

🔍

Audit logging

Every authenticated action is logged with user, timestamp, and IP for full traceability.

🛡️

Access controls

Role-based permissions and team-scoped access ensure users only see what they're allowed to.

🤖

AI data isolation

Content sent to AI features is never used to train models and is processed only for your request.

🔔

Breach notification

In the event of a breach, we notify affected users and regulators within 72 hours per GDPR.

Infrastructure Security

Julay is hosted on Amazon Web Services (AWS) in the United States. We use industry-standard security configurations for our cloud infrastructure:

  • All network traffic between services uses private VPC routing — no public endpoints for internal services.
  • Firewall rules (Security Groups) follow a least-privilege model: only required ports are open.
  • Secrets and API keys are stored in AWS Secrets Manager, never in source code or environment files.
  • Automated patching and OS updates are applied to all instances within 24 hours of release.
  • Database access is restricted to application servers; no direct public database connections exist.

Authentication & Access

  • Password hashing: All passwords are hashed with bcrypt (cost factor ≥ 12) — we never store plaintext passwords.
  • OAuth 2.0: Sign in with Google or GitHub is supported. We request only the minimum required scopes.
  • Session management: JWT tokens are short-lived (15 minutes access, 7-day refresh). Tokens are invalidated on logout.
  • Workspace isolation: All API requests are tenant-scoped. Cross-workspace data access is architecturally impossible.
  • Rate limiting: Login endpoints are rate-limited to prevent brute-force attacks.

AI Features and Data

Julay uses the Claude API by Anthropic. Content you submit to AI features is sent over an encrypted connection and is not used to train or improve Anthropic's AI models. This is guaranteed by our data processing agreement with Anthropic.

  • Only the specific content you submit when invoking an AI feature is sent to Anthropic — not your entire project history.
  • AI features are optional and clearly labeled in the interface.
  • Billing information, passwords, and authentication tokens are never included in AI requests.

Responsible Disclosure

We take security reports seriously. If you discover a vulnerability in Julay, please disclose it responsibly:

  • Email us at security@julay.org with a detailed description.
  • Include steps to reproduce, affected components, and potential impact.
  • We will acknowledge receipt within 48 hours and provide a resolution timeline.
  • We do not pursue legal action against researchers acting in good faith.
  • Critical vulnerabilities are patched within 7 days of confirmation.
